Where it is today

A working engine — not yet a product.

Being honest about the starting line is what makes the roadmap credible. Today the engine is single-tenant and builder-operated:

single tenant (one brand owner) runs on GitHub Actions cron approval = merge a Pull Request one self-hosted Postiz publisher config & secrets per repo no client dashboard yet no billing yet

It's the proof and the prototype. The next slides show exactly what turns it into a sellable SaaS.

Production-readiness · the gap

From one builder → to many clients.

🔧 Today (single-tenant)

  • One brand owner, technical, comfortable with GitHub.
  • Approval by merging a PR on the phone.
  • Per-repo brand files & GitHub secrets.
  • One Postiz instance, one set of channel keys.
  • Scheduling via repo cron workflows.

🚀 To sell it (multi-tenant SaaS)

  • Accounts, orgs & data isolation per client.
  • A friendly web/mobile approval dashboard.
  • Guided onboarding & brand-intake wizard.
  • Per-client channel OAuth + scaled publishing.
  • Billing, metering, scheduling & observability.
Production-readiness roadmap

Three phases to production-ready.

Phase 1

MVP SaaS

Make it sellable
  • Multi-tenancy + auth & orgs
  • Web/mobile approval dashboard (replaces PR-merge)
  • Onboarding / brand-intake wizard
  • Per-client channel OAuth
  • Stripe billing + basic usage metering
Phase 2

Scale

Make it reliable
  • Managed scheduler/queue (off per-repo cron)
  • Observability, retries & SLAs
  • Analytics dashboard for clients
  • White-label + custom domains
  • More channels (X, TikTok, LinkedIn)
Phase 3

Enterprise

Make it trusted
  • SSO + RBAC
  • SOC 2 / GDPR + audit trail
  • Content moderation & brand-safety
  • Cost-control budgets per client
  • Public API + brand-template marketplace
